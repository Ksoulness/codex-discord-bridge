# 更新日志

## 2026-07-21

### 新增

- 监控管理增加自动模式，可设置最近项目数和每个项目最近对话数（默认 5×5）。
- 自动模式按 Codex 最近回复顺序同步、淘汰并重排 Discord 项目分类和对话频道；淘汰只删除 Discord 副本，不删除 Codex 数据。
- 监控完整刷新统一处理分类和频道增删、排序、对话同步、状态卡与状态灯，每 10 分钟自动执行一次，手动刷新不重置定时。
- “删除停用频道”并入完整刷新；无变化的频道名、顺序、状态卡和灯不会重复写入 Discord。
- Codex 自动生成的无项目工作区分类统一显示为“无项目对话”。

## 2026-07-20

### 修复

- 修复“删除停用频道”后，发现任务使用旧映射重新创建频道的问题；取消勾选的对话不会再被自动发现流程恢复。
- 清理列表现在包含停用项目保留的频道，以及竞态遗留的未勾选活动映射，确认清理后会同时移除 Discord 频道和本地映射。

### 测试

- 增加停用项目清理、竞态残留映射清理和取消勾选后禁止重建频道的回归测试。

## 2026-07-19

### 修复

- Discord 消息写回原 Codex 对话时，如果 Desktop IPC 暂时没有会话所有者，会自动打开原对话、等待连接并重试，减少 `no-client-found` 和错误排队。
- Codex 对话信息卡不再置顶；历史信息卡会自动取消置顶，更新失败时不再新建重复卡片。
- 串行处理同一 Discord 频道的标题同步和状态灯改名，避免较早启动的标题刷新覆盖较新的黄灯、红灯或绿灯状态。

### 测试

- 增加 Desktop IPC 会话恢复、Discord 信息卡去重取消置顶，以及频道状态灯并发竞态的回归测试。
