# Discord 监控选择与频道清理设计

## 状态

本设计已由用户确认，作为后续实施计划与验收的依据。

## 背景

Bridge 当前会自动发现 Codex 对话并在 Discord 中创建项目分类和对话频道。这个行为适合全量镜像，但会把临时项目、一次性对话和不重要任务也带入 Discord，增加频道噪音。

用户只希望监控重点项目中的重点对话，并希望直接在 Discord 内完成选择、暂停和清理，不再维护额外网页或 Windows 管理程序。

## 目标

1. 在 Discord 中提供一个仅控制者可操作的 `#监控管理` 频道。
2. 以“项目 + 对话”两级结构展示 Codex 发现结果。
3. 项目启用后，仍只同步用户手动勾选的对话。
4. 新项目和新对话默认不进入 Discord。
5. 停止监控时保留 Discord 历史，清理时才删除 Discord 副本。
6. 任何管理操作都不得删除、归档或修改 Codex 原对话和项目文件。
7. 选择结果、暂停状态和清理结果在 Bridge 重启后保持一致。

## 非目标

- 不提供“项目开启后自动同步全部对话”的默认模式。
- 不在本次功能中建设独立网页、桌面托盘程序或手机 App。
- 不从 Discord 删除、归档或停止 Codex 原任务。
- 不补发停止监控期间产生的大量历史消息。
- 不改变现有对话频道中的消息镜像、审批、写回和状态灯语义。

## 方案比较

### Discord 管理频道（采用）

管理入口与监控结果位于同一服务器，手机和电脑都能使用，不需要新端口、登录页面或常驻窗口。现有 Bot 已具备斜杠命令、按钮、权限校验和 SQLite 状态存储基础。

### 本地管理网页（不采用）

可以容纳更复杂的列表和搜索，但需要额外端口、鉴权、公网或局域网访问配置，也会重新引入手机访问稳定性问题。

### Windows 管理程序（不采用）

本机操作直观，但手机无法管理，还会增加安装、开机启动和窗口隐藏维护成本。

## 用户界面

### 管理位置

Bridge 在 Discord 服务器中维护一个私有的 `#监控管理` 频道。该频道不承载 Codex 对话内容，只承载管理面板和短期交互结果。

频道权限遵循现有 `DISCORD_CONTROLLER_USER_ID`：

- Bot 可以查看、发送、编辑消息并管理 Bridge 创建的频道。
- 控制者可以查看和操作管理面板。
- 其他普通成员不能操作；在权限允许时直接隐藏该频道。

管理频道和固定面板被删除后，Bridge 启动或执行 `/codex manage` 时可以重建。管理频道自身永远不进入清理候选列表。

### 固定管理面板

固定消息显示：

- 已发现项目数。
- 已启用项目数。
- 已选对话数。
- 已暂停且可清理的频道数。
- 最近刷新时间。

面板提供以下入口：

1. `选择项目`
2. `选择对话`
3. `停止监控`
4. `清理频道`
5. `刷新列表`

Discord 每个选择菜单最多容纳 25 个选项。项目或对话超过限制时，每页最多显示 20 项，并提供上一页、下一页和返回项目按钮。对话选项显示标题，描述中包含短对话 ID 和最后活动时间，以区分重名对话。

交互结果默认使用仅操作者可见的临时回复，避免管理频道被重复操作记录刷屏。固定面板只更新汇总数字和最后刷新时间。

## 选择规则

### 项目

- 新发现项目默认关闭。
- 开启项目只代表允许从该项目选择对话，不会自动同步任何对话。
- 项目关闭时，其已选择对话全部停止监控，但选择记录保留，方便重新开启后恢复原选择。
- 项目列表显示 `项目名（已选择数/已发现数）`。

### 对话

- 新发现对话默认未选择。
- 只有“项目已开启”且“对话已选择”同时成立时，对话才允许创建或更新 Discord 镜像。
- 勾选尚未映射的对话时，Bridge 创建项目分类和对话频道，并从当前状态开始镜像。
- 勾选已有暂停频道的对话时，Bridge 复用原频道并恢复监控。
- 取消勾选只停止后续同步，不删除 Discord 历史，也不停止 Codex 任务。

### 暂停期间

停止监控后：

- 取消该对话排队中的 Discord 写入和状态改名。
- 频道标题改为白色暂停前缀，例如 `⚪-清理执行`。
- Bridge 继续推进本地来源游标，但不向 Discord 写入内容。
- 重新启用时从当前来源位置恢复，只刷新当前状态，不补发暂停期间的历史消息。

## 清理流程

`清理频道` 只列出已停止监控、未选择或项目已关闭的普通对话频道。

流程如下：

1. 用户打开清理列表并多选目标频道。
2. Bridge 显示项目、对话标题和频道数量的摘要。
3. 用户点击红色的 `确认删除 Discord 副本`。
4. 执行前再次检查这些对话仍不处于有效监控状态，即对话未选择或所属项目已关闭。
5. 对于因“项目已关闭”而暂停但仍保留原选择的对话，先明确改为未选择。
6. 删除 Discord 频道并清除对应 Bridge 映射、镜像状态和过期消息引用。
7. 项目分类中没有任何 Bridge 对话频道时，删除空分类。
8. 保留项目和对话的发现记录以及“未选择”策略，防止下一轮发现立即重建。

正在监控的频道不出现在清理列表中。清理操作永远不调用 Codex 的删除、归档、停止或写回接口。

删除 Discord 频道失败时，保留本地映射并显示失败结果，允许重试；只有 Discord 已确认目标不存在或删除成功后，才清除频道映射。

## 持久化模型

在现有 SQLite 状态库中新增三类状态。

### 项目监控策略

每个项目保存：

- `project_key`
- `project_name`
- `enabled`
- `updated_by`
- `updated_at`

### 对话监控选择与发现信息

每个发现过的普通对话保存：

- `codex_thread_id`
- `project_key`
- `thread_name`
- `selected`
- `last_seen_at`
- `updated_by`
- `updated_at`

发现信息与 `thread_bridges` 分离。删除 Discord 映射后，未选择记录仍然存在，因此自动发现不会重新创建频道。

### 管理面板位置

保存：

- Discord 服务器 ID。
- 管理频道 ID。
- 固定面板消息 ID。
- 最近成功刷新时间。

管理频道或面板消息不存在时，仅重建缺失部分并更新记录。

## 发现与镜像数据流

新的发现流程为：

```text
Codex 项目与对话发现
  -> 更新项目/对话发现记录
  -> 检查项目 enabled
  -> 检查对话 selected
  -> 仅允许命中的对话进入现有 hydrate / mirror / status 流程
```

当前静态 `discovery.allowedThreadIds` 仍作为更高优先级的运维限制：若配置了该列表，管理面板只能选择列表允许的对话。面板不能绕过本地配置安全边界。

每个对话的选择变更与镜像写入按 `threadId` 串行处理。发送 Discord 消息、改名或清理前都重新读取当前选择状态，防止“刚停止监控却又收到一条迟到消息”或“刚重新勾选却被旧清理操作删除”的竞态。

## 首次升级迁移

为避免升级后现有重点监控突然全部停止：

1. 所有已有 `thread_bridges` 记录迁移为项目已启用、对话已选择。
2. 未映射的新发现项目和对话默认关闭、未选择。
3. 迁移过程不创建、删除或改名任何 Discord 频道。
4. 用户随后在 `#监控管理` 中取消不需要的旧项目或对话，再使用清理功能删除 Discord 副本。

迁移写入版本标记，重复启动不得再次把用户已取消的对话改回已选择。

## 权限与审计

- 所有管理交互复用现有控制者身份校验，非控制者收到无权限的临时回复。
- 清理确认使用短时有效的一次性令牌，令牌绑定控制者、目标频道集合和生成时的选择版本。
- 选择、暂停、恢复、清理及失败结果写入本地 SQLite 审计记录。
- 审计只保存项目键、对话 ID、Discord 频道 ID、操作者和结果，不保存 Bot Token 或消息正文。

## 故障处理

- Discord API 暂时失败：已保存的选择不回滚；Bridge 记录错误并在下次刷新或重启时协调到目标状态。
- Codex 当前不可用：面板仍显示持久化的项目和对话，刷新提示来源不可用，不清空已有选择。
- 管理频道被手动删除：Bridge 重建管理频道和面板，不修改对话选择。
- 对话频道被手动删除：若仍被选择，Bridge 按现有映射恢复规则重建；若未选择，不重建。
- 清理过程中部分失败：逐项显示结果，成功项完成清理，失败项保留映射供重试。
- Bridge 重启：先恢复选择策略，再启动对话 hydrate 和镜像，避免启动窗口内把未选择对话重新创建出来。

## 测试范围

### 状态存储

- 新项目和新对话默认关闭、未选择。
- 选择、暂停和清理状态跨重启保持。
- 删除 `thread_bridges` 后，未选择记录仍能阻止自动重建。
- 首次迁移只执行一次，不覆盖后续人工选择。

### 选择与发现

- 项目开启但未选择对话时不创建频道。
- 只为手动勾选的对话创建或恢复频道。
- 新对话不会因所属项目已开启而自动进入 Discord。
- 静态 `allowedThreadIds` 继续限制面板可选范围。

### 暂停与恢复

- 取消勾选后不再发送消息或改写任务状态。
- 暂停频道显示白色前缀。
- 暂停期间推进来源游标，恢复后不补发旧消息。
- 活动任务被暂停不影响 Codex 本身继续执行。

### 清理

- 只列出未监控频道。
- 二次确认前重新校验选择状态。
- 清理删除 Discord 副本和 Bridge 映射，但不调用任何 Codex 删除或归档接口。
- 空项目分类被删除，非空分类保留。
- Discord 删除失败时本地映射保留。

### Discord 交互

- 只有控制者可以操作。
- 超过 25 个项目或对话时分页正常。
- 管理频道和固定面板可在被删除后恢复。
- 重复点击、过期按钮和并发操作不会重复创建或误删频道。

## 验收标准

1. Discord 中存在可从手机和电脑操作的私有 `#监控管理` 频道。
2. 项目开启后，只有手动勾选的对话进入 Discord；所有新对话默认不进入。
3. 停止监控后 Discord 历史保留，频道变为白色暂停状态，且不再接收更新。
4. 恢复监控后从当前状态继续，不补发暂停期间的历史。
5. 清理列表只包含未监控频道，并在二次确认后删除 Discord 副本及映射。
6. 清理、暂停和选择操作不会删除、归档、停止或修改 Codex 原对话和项目文件。
7. 选择结果在 Bridge 重启和开机启动后保持。
8. 现有镜像、审批、写回、状态灯和频道中文命名回归测试全部通过。
